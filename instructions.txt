### Typography System

**Primary Typeface (Headlines):** Space Grotesk, Inter, or custom futuristic grotesk
- Weight: 600–800
- Tracking: +0.5% to +2% (wide for authority)
- Text rendering: Crisp, anti-aliased

**Secondary Typeface (Body/UI):** Inter, Helvetica Neue, or Satoshi
- Weight: 400–500
- Line height: 1.6–1.8 (readability over density)

**Monospace (Telemetry/Stats):** IBM Plex Mono, Courier Prime, or Roboto Mono
- Used for: KPIs, uptime stats, system coordinates, code snippets
- Weight: 500–600

### Language Patterns

**DO:**
- Use active, present-tense language ("Command the network" not "The network can be managed")
- Short, punchy headlines (max 8 words)
- Technical confidence ("Sub-millisecond latency" not "Really fast")
- Avoid corporate jargon

**DON'T:**
- Use empty buzzwords ("synergy," "leverage," "best-in-class")
- Soften with qualifiers ("arguably," "perhaps," "we believe")
- Default to passive voice
- Oversimplify technical concepts

## CORE DESIGN SYSTEMS

### Motion Principles

**Scroll Behavior (Critical)**

- **No traditional scrolling:** Every pixel scrolled = time coordinate advancing through 3D space
- Scroll speed modulates camera movement depth
- Fast scroll = rapid zoom; slow scroll = precise navigation
- Parallax ratio: 0.3x–1.5x depending on layer depth

**Animation Timing**

| Element Type | Duration | Easing | Purpose |
|---|---|---|---|
| Micro-interactions (hover) | 200–300ms | cubic-bezier(0.34, 1.56, 0.64, 1) (bounce) | Feedback |
| Section transitions | 800ms–1.2s | ease-in-out | Cinematic feel |
| Data reveals | 400ms–800ms | ease-out | Intelligence |
| System initialization | 1.5s–3s | custom sequence | Immersion |

**Cursor Behavior**

- Default state: Minimalist crosshair with pulsing dot
- Hover on interactive elements: Morphs into radar/signal icon with rotating rings
- Hover on text: Scans left-to-right with neon underline
- Click state: Signal pulse expands outward

### Interaction Layers

1. **Foreground (UI):** Buttons, text, interactive elements (opacity: 1, z-index high)
2. **Mid-layer (Context):** Animated diagrams, icons, data visualizations (opacity: 0.6–0.8)
3. **Background (Environment):** 3D grid, particles, ambient motion (opacity: 0.2–0.4)
4. **Deep Background (Atmosphere):** Subtle gradients, far-field lights (opacity: 0.1)

0.6s  → Nodes begin pulsing (staggered)
0.8s  → Particles activate + begin flowing
1.2s  → Camera begins forward movement
1.5s  → Hero text reveals (character-by-character scanning effect)
2.5s  → Subtitle fades in
3.2s  → CTA buttons glow active
```

CONTROL THE FUTURE.
```

Styling:
- Main headline: 72px–96px, 800 weight, letter-spacing +1.5%

**Subheading (below, smaller):**

Next-generation telecom control platform for 
real-time intelligence, autonomous operations,

## TECHNICAL ANIMATION SPECIFICATIONS FOR DEVELOPERS

### Required Libraries & Engines

```
Primary: Three.js (r128+) for 3D environments
Animation: GSAP (latest) for timeline-based animations
Scroll: Locomotive Scroll or custom scroll-hijacking system
UI: Tailwind CSS + custom CSS for real-time effects
Interactions: Cursor.js or custom cursor library
```

### Scroll-to-Animation Mapping (Critical)

```javascript
// Pseudo-code
scrollPos (0–100%) → 3D camera movement (Z-axis)
scrollVelocity → animation speed multiplier
mouseX → parallax depth modifier
mouseY → parallax depth modifier

// Example:
if (scrollPos > 20% && scrollPos < 40%) {
  // Hero Section Parallax
  backgroundLayer.position.z += scrollDelta * 0.5;
  midLayer.position.z += scrollDelta * 1.0;
  foregroundLayer.position.z += scrollDelta * 1.5;
}
```

### Cursor Morphing Rules

```javascript
// Default
cursor = crosshair + pulsing dot

// On hover interactive
cursor = radar icon (rotating)

// On hover text
cursor = scanning underline (left-to-right animation)

// On click
cursor = expanding signal pulse
```

### Color & Opacity Feedback

```css
/* Interactive element hover state */
--primary: #FF6A00;
--glow: 0 0 20px #FF6A00, inset 0 0 10px rgba(255, 106, 0, 0.3);
--scale: 1.05;
--brightness: 1.2;
```

## RESPONSIVE DESIGN STRATEGY

**Mobile (< 768px):**
- Disable full 3D scenes (show static high-res images instead)
- Simplify parallax to opacity-only transitions
- Stack all content vertically
- Keep hero video/animation but reduce complexity
- Mobile-optimized cursor effects (tap states)

**Tablet (768px–1200px):**
- Reduced particle counts
- Simpler 3D models
- Slower animation timing
- Maintain full feature set

**Desktop (> 1200px):**
- Full 3D environments
- Complex particle systems
- Advanced scroll interactions
- Premium visual effects

**Technical:**
- Page load: < 3.5s (with partial content)

- Demo request CTR: > 8%

## FINAL DESIGN PRINCIPLES

1. **Movement Before Messaging** — Let visuals tell the story first; copy reinforces.
2. **Authenticity Over Flashiness** — Visual effects serve function, not just aesthetics.
3. **Technical Credibility** — Every animation, stat, and claim must feel substantive.
4. **Spatial Storytelling** — Use depth and parallax to build narrative.
5. **Responsive Immersion** — Experience scales beautifully across all devices.
6. **Performance is Feature** — Smooth 60fps animation is a feature, not a luxury.
7. **Trust Through Precision** — Exact specs, no guessing; hand-off should be seamless.
